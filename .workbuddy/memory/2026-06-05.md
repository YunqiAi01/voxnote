### Bug 修复：登录后支付跳转循环
- **问题**：用户从定价页点击订阅 → checkout API 重定向到 `/login?redirect=/pricing` → 登录后总是跳到 `/` 首页 → 用户再点支付又跳回登录（死循环）
- **根因**：`login/page.tsx` 和 `register/page.tsx` 忽略 URL 中的 `redirect` 参数，硬编码跳转到 `/`
- **修复**：
  1. 两个页面都改为使用 `useSearchParams()` 读取 `redirect` 参数
  2. 登录/注册成功后导航到 `redirect` 指定的页面（默认为 `/`）
  3. 登录页的"注册"链接和注册页的"登录"链接都保留 `redirect` 参数
  4. 使用 `Suspense` 包裹 `useSearchParams`（Next.js 要求）
- **验证**：构建成功，零 TypeScript 错误，已推送到 GitHub → Vercel 自动部署

### Bug 修复：定价页按钮点击没反应（2026-06-08）
- **问题**：点击"立即抢购"/"订阅月付"等按钮完全没有反应
- **根因**：定价页使用 `"use client"` + `useSearchParams()` + `Suspense` 模式，内容完全依赖客户端 JS 渲染。如果 hydration 出问题，按钮根本不在 HTML 中
- **修复**：重写为**服务端组件**（async function），用 Next.js 页面级的 `searchParams` prop 替代 `useSearchParams()` hook
  - 移除 `"use client"`、`useSearchParams`、`Suspense`
  - pricing 页面改为 `ƒ (Dynamic)` — 服务端按需渲染，确保定价卡片和按钮始终在 HTML 中
  - `Header` 作为客户端组件嵌套在服务端组件中正常工作
- **部署**：已构建验证通过，推送到 Vercel 自动部署
